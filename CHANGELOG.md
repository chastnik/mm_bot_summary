## 🚀 [v2.4] - 2025-07-16

### ✨ Новые возможности

#### 🎯 Расширенная поддержка форматов команд
- **Максимально естественные команды**: Теперь поддерживаются самые простые форматы
- **Новые форматы**:
  - `~канал каждую среду в 6 вечера`
  - `~канал вторник 18:00`
  - `~канал пятница 17:30`
  - `~канал четверг 14:00`

#### 🔧 Улучшенные парсеры
- **Парсер времени**: Поддержка времени без предлога "в" (`18:00` вместо `в 18:00`)
- **Парсер дней недели**: Все формы дней недели (`среда`, `среду`, `средам`)
- **Парсер частоты**: Автоматическое определение еженедельной частоты по дню недели
- **Умное распознавание**: Команды определяются по наличию каналов + время + день недели

#### 📝 Обновленная документация
- **Примеры всех форматов** в README.md и справочных сообщениях
- **Пошаговые инструкции** для пользователей
- **Расширенные примеры** в документации

### 🔄 Поддерживаемые форматы команд:

```bash
# Расширенные форматы
~general каждую среду в 6 вечера
~development вторник 18:00
~marketing пятница 17:30
~support понедельник в 9 утра

# Классические форматы
~general, ~random ежедневно в 9 утра
~development, ~qa еженедельно по вторникам в 18:00

# Старый формат (поддерживается)
general,random ~ 09:00 ~ daily
```

### 🎨 Примеры использования:

**Простые команды:**
- `~general пятница 17:30` → еженедельно по пятницам в 17:30
- `~support вторник 10:00` → еженедельно по вторникам в 10:00

**Естественные команды:**
- `~marketing каждую среду в 6 вечера` → еженедельно по средам в 18:00
- `~development каждый понедельник в 9 утра` → еженедельно по понедельникам в 09:00

### 🔄 Обратная совместимость:
- Все существующие подписки продолжают работать
- Поддержка старых форматов команд
- Плавный переход для пользователей

## 🚀 [v2.3] - 2025-07-16

### ✨ Новые возможности

#### 📅 Поддержка дней недели для еженедельных подписок
- **Гибкое планирование**: Указание конкретного дня недели для еженедельных подписок
- **Новый формат**: `~канал1, ~канал2 еженедельно по вторникам в 18:00`
- **Все дни недели**: Поддержка всех дней недели с различными формами написания
  - `по понедельникам`, `по вторникам`, `по средам`, `по четвергам`
  - `по пятницам`, `по субботам`, `по воскресеньям`

#### 🎯 Точный расчет периодов
- **Умный сбор сообщений**: Для еженедельных подписок по средам в 15:00 собираются сообщения с прошлой среды 15:00 по текущую среду 15:00
- **Ежедневные подписки**: Сбор за последние 24 часа от времени подписки
- **Фильтрация по времени**: Точная фильтрация сообщений по указанному периоду

#### 🗄️ Обновленная база данных
- **Новое поле weekday**: Хранение дня недели (0 = понедельник, 6 = воскресенье)
- **Автоматическая миграция**: Плавное обновление существующих подписок
- **Обратная совместимость**: Старые подписки продолжают работать

#### 📊 Улучшенное отображение
- **Веб-интерфейс**: Отображение дня недели в списке подписок
- **Личные сообщения**: Показ дня недели в команде "подписки"
- **Подтверждения**: Указание дня недели при создании подписки

### 🔧 Улучшения

#### Парсинг команд:
- **Распознавание дней недели** в естественном формате
- **Гибкие формы**: `по средам`, `по пятницам`, `средам`, `пятницам`
- **Сокращения**: `пн`, `вт`, `ср`, `чт`, `пт`, `сб`, `вс`

#### Планировщик:
- **Проверка дня недели** для еженедельных подписок
- **Точный расчет периодов** с учетом часового пояса
- **Улучшенная фильтрация** сообщений по времени

### 📋 Примеры новых команд:

```bash
# Еженедельные подписки с днями недели
~general, ~random еженедельно по понедельникам в 9:00
~development, ~qa еженедельно по средам в 18:00
~marketing каждую неделю по пятницам в 15:30

# Ежедневные подписки (без изменений)
~support ежедневно в 10:00
```

### 🔄 Обратная совместимость:
- Все старые подписки продолжают работать
- Еженедельные подписки без указания дня недели работают как раньше
- Автоматическая миграция базы данных
